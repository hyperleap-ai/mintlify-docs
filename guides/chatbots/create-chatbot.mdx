## Prerequisites

Before creating Chatbots, ensure you have:

- Access to [Studio](https://studio.hyperleapai.com)
- An active Chatbot subscription
- Website access for deployment

<Note>
  You have to [whitelist](https://studio.hyperleapai.com/manage-organization)
  the domain of your website in the organization settings to use the chatbot.
</Note>

    ### Programmatic Control

    You can control the chatbot from your website using custom buttons or links:

    <Tabs>
      <Tab title="Data Attributes (Easiest)">
        Add the `data-aichatbot-invoke` attribute to any HTML element:

        ```html
        <!-- Open chatbot on click -->
        <button data-aichatbot-invoke="open">Ask AI Assistant</button>

        <!-- Close chatbot on click -->
        <button data-aichatbot-invoke="close">Close Chat</button>

        <!-- Toggle chatbot on click -->
        <a href="#" data-aichatbot-invoke="toggle">Chat with us</a>

        <!-- Empty attribute defaults to toggle -->
        <div data-aichatbot-invoke>Need Help?</div>
        ```

        Elements with this attribute are automatically bound when the chatbot loads. No additional JavaScript required!
      </Tab>

      <Tab title="JavaScript API">
        Use the global `HyperleapChatbot` API for custom control:

        ```javascript
        // Open the chatbot
        window.HyperleapChatbot.open();

        // Close the chatbot
        window.HyperleapChatbot.close();

        // Toggle the chatbot
        window.HyperleapChatbot.toggle();
        ```

        Example usage with custom button:

        ```html
        <button id="my-custom-button">Get Help</button>

        <script>
          document.getElementById('my-custom-button').addEventListener('click', function() {
            window.HyperleapChatbot.open();
          });
        </script>
        ```
      </Tab>
    </Tabs>

    <Info>
      **Features:**
      - Works with any HTML element (buttons, anchors, divs, etc.)
      - Supports dynamically added elements
      - Automatically prevents default actions on links
      - No duplicate event listeners
    </Info>
